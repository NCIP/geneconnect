      Title : Advanced Search for single query

      VITALS
      Product GeneConnect
      Component WebInterface
      		 Test Area Advanced Search
      Reference 1 Priority P1

      DESCRIPTION

      		 
      Purpose:
 
To check Advanced Search functionality with more than one genomic identifier for single query.

Setup:

1. Netscape 7/Mozilla Firefox 1.5/Internet Explorer 6 must be installed.
2. For Safari browser follow the below steps:
    b. Safari browser requires WebKit web browser engine 
       http://webkit.org.
    c. Install WebKit and open Safari browser through WebKit 
       http://webkit.org/building/run.html.  

Procedure:

1. Go to the GeneConnect Home Page by clicking on this link:
http://128.252.178.209:9092/geneconnect/Home.do

2. On the left side of the page click on the ADVANCED SEARCH link.This would open the Advanced Search page.

3. There are three parts on the page.
   a> Select Input Data Sources (spreadsheet view listing data source name as header and text box to enter genomic identifer).
   b> Enter Path score (text box to enter predicate for path score).
   c> Select Output Data Sources (test boxes, one for each data souce to enter predicate for frequency).

4. In the Input Data Sources section enter the genomic identifier value for one or more data sources.
   
5. Next select the output data source/s by checking in against the appropriate data source/s from the list available at the bottom of the page. On selecting a check box the frequency text box should get enable. If check box is not selected the frequency text box should be disabled.

6. Click on search button to view the result.

Input/Data:



Expected Output:

1. The Advanced Search result page should be displayed.
2. The result page should contain the input genomic identifier with the desired output genomic identifier/s.
3. The combo box and text area with title 'View result display' should appear on top listing entered input queries. 
4. The 'All' should get selected in combo box and same should get entered in text area.
5. By default result page should display result of all queries.
6. If no matching entry is found a "No Match Found" message should appear.
7. If more than one output data source is selected and a matching entry is not found for some of them, then a "No Match Found" would appear against them and entries would appear against the rest.
8. The output genomic identifiers displayed should be matching with the input ID selected.
9. Clicking on Path icon should display the graph showing only one or more paths selected as searfh criteria.  


Verification Logic:
1. Go to the input data source/s webpage. Example-If Entrez Gene is selected as one of the input data source, go to the Entrez Gene homepage.
2. Search with the ID you have entered in the Advanced Search page.
3. If direct annotation to the selected output source exists for the input ID, then the output ID from the webpage and that in the Simple Search  result page should match.
4. For Path and Frequency score verification please refer to 
verification logic of 'Simple Search Frequency and Path Score' 
test case.

Result Data:

Please refer to Excel file for results for query EntrezGene=1958 and EntrezGene=1959 and all data sources as output
http://cabigcvs.nci.nih.gov/viewcvs/viewcvs.cgi/geneconnect/Coding%20and%20Testing/Test%20Activity%20Logs/SearchResult.xls

      Purpose:

To test the GeneConnect Simple Search Interface to retrieve associated genomic identifiers for a single locus, with one/more Input Data Source and corresponding genomic identifier?s provided as input.

Setup:

Netscape 7/Mozilla Firefox 1.5/Internet Explorer 6 must be installed.

Procedure:

1. Follow the link: http://128.252.178.209:9092/geneconnect/Home.do and click on 'Simple Search'.

2. Select Input Data Source from the 'Data Source' drop down and type in the genomic identifier corresponding to the Data Source in the area provided.

4. Select one/more Output Data Source by checking the corresponding check boxes.

5. Click the "Search" button at the bottom of the page.

Input/Data:

1. Genomic identifier for all the data sources selected as the 'Input Data Source' from the drop down.
(Selection of at least one Input Data Source is mandatory.)

      Test Case ID : 2279
      Short Title : Frequency and Path Score validation
      Title : Frequency and Path Score validation

      VITALS
      Product GeneConnect
      Component WebInterface
      		 Test Area Simple Search
      Reference 1 Priority P1

      DESCRIPTION

      		 
      Purpose:

To check calculation and visualisation of frequency and Path score result page.

Setup:



Procedure:
1. Go to the GeneConnect Home Page by clicking on this link:
http://128.252.178.209:9092/geneconnect/Home.do

2. Follow the steps as given in Simple Search  with one identifier test case.

3. On the result page check in the frequency and path score check boxes on the top of the page.

Input/Data:



Expected Output:
1. The Path score column for the each result set should be visible.
2. The Frequency column for each result set should be visible. The 
   Frequency column is applicable to only selected output data sources.

Verification Logic:
1. Path score is defined as the ratio of number of occurrences of a 
   unique ?combination of genomic identifiers? in a query, to the total 
   number of traversable paths for the given input. Path score is 
   calculated for a set of genomic identifiers and a given ONT
   The Path score would be verified as follows:
   a) For each result 
       i) Click on 'show graph' magnifier icon. This will open a window 
          showing selected ONTs for this result.
       ii) Count the number of ONT for this set, The count is given as 
           suffix to each path. 
   b) Count total number of ONTs i.e number of ONTs considered for all   
      set.
   c) Path score for each result set should be number of ONTs associated 
      with that result divided by total number of ONT.
2. Frequency is defined as the ratio of number of occurrences of 
   a ?single genomic identifier? in each combination of genomic 
   identifiers to the total number of traversable paths for the given 
   input.
   The frequency would be verified as follows:
   a) Count number of ONTs associated for 'single genomic identifier'. 
      For example consider Entrez Gene Id '1958'. Count ONTs associated 
      with set where value Entrez Gene Id is '1958'.
   b) Frequency for that genomic identifier should be 
      number of ONTs associated with that genomic identifier divided by 
      total number of ONT in the result set.
