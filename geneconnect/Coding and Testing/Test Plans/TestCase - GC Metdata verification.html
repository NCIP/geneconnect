      Test Case ID : 1893
      Short Title : Metadata Calculation
      Title : Testing Metadata Calculations

      VITALS
      Product GeneConnect
      Component Metadata Calculation
      		 Test Area Metadata Calculation
      Reference 1 Priority P2

      DESCRIPTION

      		 
      Purpose:
To check that the metadata tables are created and populated with the correct data.

Setup:
GeneConnect Server has been deployed on the machine (Deployment Target of the GeneConnect Build.xml has been run).

Procedure:

1.Configure and Verify  the file GCGraph.txt in \config folder.
2.Configure and Verify  the file server.properties in \config folder.
3.Run the Deploy target of GeneConnect Build.xml.


Input/Data:
Configuration file representing GeneConnect graph (GCGraph.txt) located at \Config  folder.

Expected Output:

The following metadata tables must be loaded with correct data:
DATASOURCE
DATASOURCE_LINKS
BASETABLE_METADATA
COLUMN_METADATA
ONT
PATH
PATH_ONT
SUBPATH
TABLEJOIN_METADATA

Verification Logic:

This test case is written with reference to an example path, but can be applied to all paths/ONT's.
Consider the following path:
Path1: Ensembl Gene - Entrez Gene - RefSeqmRNA - Ensembl Transcript.

The Metadata tables can be verified as follows:
1.To verify that the table DATASOURCE is correctly loaded:
All the datasources as mentioned in the GCGraph.txt must be present in the table DATASOURCE and must have a unique DATASOURCE_ID.
Also corresponding CLASS and TABLE_NAME must be specified as GENE, MRNA or PROTEIN.

2.To verify that the table DATASOURCE_LINKS is correctly loaded:
(i)All the adjacent links (as per the GCGraph) are stored in the table DATASOURCE_LINKS. Thus , for the DataSource ,Ensembl Gene, all the adjacent links are :
Ensembl Gene - Entrez Gene
Ensembl Gene - UniGene
Ensembl Gene - Ensembl Transcript

(ii)All of these are represented in DATASOURCE_LINKS as follows:
For the 1st link Ensembl Gene - Entrez Gene, Ensembl Gene is the Source and Entrez Gene is the destination. The linktype is direct. Thus, the entry in the table would be SOURCE_DATASOURCE_ID as 0,TARGET_DATASOURCE_ID as 1 and LINK_TYPE_ID as 1.
Similar logic applies for all the links.


3. To verify that the table BASETABLE_METADATA is correctly loaded.
(i)Find the tables involved in path1 as follows
Table 1:ENSEMBLGENE_ENTREZGENE
Table 2:ENTREZGENE_REFSEQMRNA
Table 3:REFSEQMRNA_ENSEMBLTRANS

(ii)For Table 1:ENSEMBLGENE_ENTREZGENE, Ensembl Gene becomes the 'SOURCE data source' and Entrez Gene becomes the 'TARGET data source?. This is applied for all the tables.

(iii)For Table 1:ENSEMBLGENE_ENTREZGENE, find 'DATASOURCE_ID' from the table DATASOURCE for Ensembl Gene and Entrez Gene. These 'DATASOURCE_ID's must be present in the table BASETABLE_METADATA in the columns SOURCE_DATASOURCE_ID and TARGET_DATASOURCE_ID respectively corresponding to the table name 'ENSEMBLGENE_ENTREZGENE'.

Similar procedure must be followed for the tables ENTREZGENE_REFSEQMRNA and REFSEQMRNA_ENSEMBLTRANS.

4. To verify that the table COLUMN_METADATA is correctly loaded.

(i)For Table 1:ENSEMBLGENE_ENTREZGENE, find the following:
-TABLE_ID from the table BASETABLE_METADATA for the table ENSEMBLGENE_ENTREZGENE
-DATASOURCE_ID from DATASOURCE table for the two data sources in this table (Ensembl Gene and Entrez Gene).

(ii)These values must be present in the table COLUMN_METADATA in the columns TABLE_ID and DATASOURCE_ID respectively corresponding to the 3 columns (ESE_ENSEMBLGENEID, ESE_GENEID, ESE_LINKTYPE). 
Here, the TABLE_ID would be the same for all the 3 columns.
Note that ESE_LINKTYPE column would have no DATASOURCE_ID since it doesn?t represent a Data Source.

Similarly procedure must be followed for the tables ENTREZGENE_REFSEQMRNA and REFSEQMRNA_ENSEMBLTRANS.

5. To verify that the table TABLEJOIN_METADATA is correctly loaded.
(i)For the path1: Ensembl Gene - Entrez Gene - RefSeqmRNA - Ensembl Transcript, the 3 tables involved are: 
1. ENSEMBLGENE_ENTREZGENE
2. ENTREZGENE_REFSEQMRNA
3. REFSEQMRNA_ENSEMBLTRANS

(ii)Here the tables 1 and 2 have ENTREZGENE column in common. While relating table 1 and 2, table1 becomes Source table and table 2 becomes Target table. 
Find the following:
-TABLE_ID for Source and Target table from the table BASETABLE_METADATA
-Find the COLUMN_ID from table COLUMN_METADATA for the column which is common (Here Entrez Gene) in both the tables (1 and 2).

(iii)These values for the TABLE_ID for Source and Target table must be present in the table TABLEJOIN_METADATA as columns SOURCE_TABLE_ID and TARGET_TABLE_ID respectively.
The values for COLUMN_ID for table 1 and table 2 must be present in the table TABLEJOIN_METADATA as columns SOURCE_COLUMN_ID and TARGET_COLUMN_ID respectively.

Similar procedure must be followed for the tables ENTREZGENE_REFSEQMRNA and REFSEQMRNA_ENSEMBLTRANS.

6. To verify that the table PATH is correctly loaded: 
(i)For each of the Data Source involved in the path1, find the DATASOURCE_ID from the table DATASOURCE as follows:
Ensembl Gene - Entrez Gene - RefSeqmRNA - Ensembl Transcript.
Path is: 0_1_5_4

(ii)This path should be present in table PATH with 
SOURCE_DATASOURCE_ID as 0
TARGET_DATASOURCE_ID as 4
PATH (intermediate nodes) as 1_5

7. To verify that the table ONT and PATH_ONT is correctly loaded:
(i)Find the PATH_ID  for the path 0_1_5_4 from table PATH.

(ii)Find associated ONT_ID from PATH_ONT table for the path 0_1_5_4.
Lets assume the values 28, 32 as ONT_ID?s . 

(iii)For PATH_ID 28 and 32, from table ONT, the corresponding SOURCE_DS_ID must have 0, and the subsequent records must have the SOURCE_DS_ID as 1, 5, and 4 respectively indicating the data sources in the path.

(iv)The difference in the PATH_ID's 28 and 32 is depending on the link types involved in the same path. This difference is shown by the column LINKTYPE_ID, which has value 2 (inferred link) for link between SOURCE_DS_ID's 5 and 4 for PATH_ID 28 and value 4 (BLAST Identity link) for PATH_ID 32.

(The values for link types are referred from the table LINK_TYPE with columns LINK_TYPE_ID and LINK_TYPE_NAME.)


8. To verify that the table SUBPATH  is correctly loaded:
(i)All the sub paths of  a particular path can be referred in table SUBPATH. For PATH_ID 28, find the corresponding SUBPATH_ID in table SUBPATH. Here, for PATH 28, the SUBPATH_ID's are : 34,48,78,84,93,331,341,427. 

(ii)These ID's indicate the sub paths which can be verified as follows: 
Check the SUBPATH_ID value (34) in PATH table, in column PATH_ID.This entry shows the sub path 0-1, indicated by 0 in SOURCE_DATASOURCE_ID and 1 in TARGET_DATASOURCE_ID.

(iii)Similarly for PATH_ID : 48 in PATH table in column PATH_ID , shows the sub path 0-1-5 , indicated by 0 in SOURCE_DATASOURCE_ID, 5 in TARGET_DATASOURCE_ID and 1 in PATH (intermediate node).

All the other sub paths can be verified with similar logic.

      Test Case ID : 1896
      Short Title : Metadata Calculation for newly added DataSource
      Title : Testing Metadata Calculations for newly added DataSource

      VITALS
      Product GeneConnect
      Component Metadata Calculation
      		 Test Area Metadata Calculation
      Reference 1 Priority P2

      DESCRIPTION

      		 
      Purpose:
To test whether the newly added Data Source is correctly represented in the relevant Metadata Tables.

Setup:
1.GeneConnect Server has been deployed on the machine (Deployment Target of the GeneConnect Build.xml has been run).

Procedure:

1.Check that the added Data Source has been added in the file GCGraph.txt in \Config folder.
2.Verify that the adjacency matrix is calculated accordingly.
Each entry in the matrix represents the link types between two Data Sources. For e.g if we are calculating the matrix value for Ensembl Protein to UniprotKB, then 
Link types = Alignment based + Inferred = 12 + 2 = 14.
Thus there should be value 14 for in the adjacency matrix for the directional link from Ensembl Protein to UniprotKB.
Similarly the adjaceny matrix values can be calculated for added data source depending on the links by which it is related to other data sources.
Legend:
Alignment based->12
Direct->1
Inferred->2
3.Verify  that the file is present in GCGraph.txt in \config folder.
4.Configure and Verify  the file server.properties in \config folder.
5.Modify the SQL scripts GCSchemaCreation.sql for defining the base tables for new data source and GCMetadataPopulation.sql  to define the metadata for those base tables.
Also modify GCMetadataSchemaCreation.sql for adding the column in GENE, MRNA or PROTEIN table.
6.Run the Deploy target of GeneConnect Build.xml.

Input/Data:
Configuration file representing GeneConnect graph (GCGraph.txt) located in \Config  folder.This file must also have the added Data Source.

Expected Output:
The following metadata tables must be loaded with correct data including the data for the added Data Source:
DATASOURCE
DATASOURCE_LINKS
BASETABLE_METADATA
COLUMN_METADATA
ONT
PATH
PATH_ONT
SUBPATH
TABLEJOIN_METADATA

Verification Logic:


1.Verify the entries for the newly added data source in the following tables:
(i) The added DataSource must be present in the table DATASOURCE with a unique DATASOURCE_ID. Other attributes like DATASOURCE_NAME, CLASS, ATTRIBUTE, ATTRIBUTE_TYPE, TABLE_NAME, and COLUMN_NAME should be specified. 

(ii).Depending on the Data Source ,corresponding CLASS and TABLE_NAME is specified as Gene, mRNA or Protein in the table DATASOURCE_ID.

2.Correspondingly verify the other tables by applying the verification logic described in the  test case 'Metadata Calculation' .
