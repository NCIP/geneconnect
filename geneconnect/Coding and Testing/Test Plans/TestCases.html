      Purpose:
To ensure that the Ensembl Transcript files are parsed to get formatted Ensembl fasta files for Blast by testing the test input file provided with the deliverable.

1. Verify following entries in UpdateLinux.sh:

3. Check the server.properties file (in the 'Config' folder) and verify the database driver name and the dbURL and other database arguments including the TNS service name for the Oracle database, which can be any one of the following:
a) oracle.jdbc.driver.OracleDriver : for Driver name
b) jdbc:oracle:oci8:@<database name> : for database URL

Ensure that the external parser's .sh file is present in the 'ParserScripts' folder.

1.EnsemblGene_EnsemblTranscript_Hs.txt

Verification Logic:

1.Check the correctness of the data loaded into tables by matching all the Ensembl Gene base tables with the expected output files as listed below:

TABLE EXPECTED-----------------------------------OUTPUT FILE

ENSEMBLGENE_ENSEMBLTRANSCRIPT----->EnsemblGene_EnsemblTranscript_Hs.txt
ENSEMBLTRANSCRIPT_ENSEMBLPROTEIN-->EnsemblTranscript_EnsemblPeptide_Hs.txt
ENSEMBLGENE_ENTREZGENE------------>EnsemblGene_EntrezGene_Hs.txt

ENSEMBLTRANSCRIPT_REFSEQMRNA------>EnsemblTranscript_RefSeqmRNA_Hs.txt
ENSEMBLPROTEIN_REFSEQPROTEIN------>EnsemblPeptide_RefSeqProtein_Hs.txt
ENSEMBLPROTEIN_UNIPROT----->EnsemblPeptide_UniProKBPrimaryAccession_Hs.txt
