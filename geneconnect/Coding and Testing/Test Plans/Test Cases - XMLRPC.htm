      Test Case ID : 1661
      Short Title : Blastp - Refseq Protein vs. Ensembl peptide
      Title : Blast against a protein sequence to verify the functionality of the XMLRPC system

      VITALS
      Product GeneConnect
      Component XMLRPC System
      		 Test Area System Tests
      Reference 1 Priority P1

      DESCRIPTION

      		 
      Purpose:
To ensure that the XMLRPC system works with blast against a protein sequence.

Setup:
1. Ensure that the following are set up/installed for Perl and to obtain sequence alignement results:

        Perl (CPAN)
		bioperl
		bioperl-run
		libwww
		URI
		DBM::Deep
		Fontier-RPC

	Sequence Alignment
		needle
		blastn
		blastp

2. The following custom modules should be present in Perl:

		Cluster::Job
		Cluster::JobQueue
		Cluster::Machine
		Cluster::MachineQueue
		Cluster::QueueManager
		GeneConnect::Hit
		GeneConnect::Params
		SplitSeqFile.pm

3. The target database (Ensembl's Homo_sapiens.NCBI36.apr.pep.fa) should be set up as a BLAST database on the compute node (compute-0-16).  The path to the database (/state/partition1/blastDB/) should be specified in the environment variable BLASTDB.

4. Copy the following files to /home/clususer/geneconnect/XML_RPC_System/test
		SplitSeqFile.pm
		split_fasta_file.pl
		XML_RPC_Server.pl
		XML_RPC_Server_Monitor.pl
		XML_RPC_testclient.pl
		machines.txt
		fastarefprotein10.fasta (containing query sequences)
5. Copy the following files to /home/clususer/geneconnect/XML_RPC_System/test/Cluster
		Job.pm
		JobQueue.pm
		Machine.pm
		MachineQueue.pm
		QueueManager.pm
6. Copy the following files to /home/clususer/geneconnect/XML_RPC_System/test/GeneConnect
		Hit.pm
		Params.pm
7. Copy gc_seq_align_pipeline.pl to the compute node (compute-0-16), in /state/partition1/geneconnect
8. Set the configuration parameters in XML_RPC_testclient.pl to the following:

	SRCFILE => 'fastarefprotein10.fasta',
	DESTINATIONDB => 'Homo_sapiens.NCBI36.apr.pep.fa',
	PATHS => {
	 	    db => '/state/partition1/blastDB/',
		base => '/home/clususer/geneconnect/XML_RPC_system/test/',
	      source => '/home/clususer/geneconnect/XML_RPC_system/test/',
	      output => '/home/clususer/geneconnect/XML_RPC_system/test/',
		  },
	      BLAST => {
			  blastType => 'blastp',
			  EValue => 1e-25,
			  ScoringMatrix => 'BLOSUM62',
			  GapOpenCost => undef,
			  GapExtensionCost => undef,
			  Dropoff => undef,
			  WordSize => undef,
			  seqType => 'protein',
			},
			BLAST_FILTER => {								     PercentSequenceAlignment => 0.80,
				},
			NEEDLE => {
				GapOpenCost => undef,
		                GapExtend => undef,
		                Matrix => undef,
		              },
			NEEDLE_FILTER => {
		                 	PercentSequenceAlignment => 0.90,
			                 	SimilarityScore => 0.90,
			                 	IdentityScore => 0.90,
		                 		NumberMisMatch => 2,
		                 	},
			CONFIG => {
				numSeqsPerJob => 3,
		                saveAlignOutput => 1,
		                controlFileHeader => 'test header',
		                controlFileColumnSeparator => '###',
		              },

9. The machines.txt file should contain the name of the compute node (compute-0-16).

10. In XML_RPC_Server.pl, verify the following :
		a. the master node name (im-vishnu) is provided to initialize_queues(), 'db_queue'
		b. in seq_align_pipeline(), the path to split_fasta_file.pl (on the master node) is correct
		c. in do_alignments(), the path to gc_seq_align_pipeline.pl (on the compute nodes) is correct.

Procedure:
1. Start the XML_RPC_Server.
2. (optional) In a separate terminal, start the XML_RPC_Server_Monitor.
3. In a separate terminal, execute XML_RPC_testclient.pl.

Input/Data:
Homo_sapiens.NCBI36.apr.pep.fa (target database)
fastarefprotein10.fasta (containing query sequences)

Expected Output:

1. A working directory will be created in the base path (/home/clususer/geneconnect/XML_RPC_system/test/).  The working directory is named YYYYMMDD_(query sequence filename)_(target database), where YYYYMMDD corresponds to the current year, month, and day and the query sequence and target database names are as specified in the configuration parameters.  The working directory for this test should be named 'YYYYMMDD_fastarefprotein10.fasta_Homo_sapiens.NCBI36.apr.pep.fa', where YYYYMMDD is the current date.
	2. The following files will be created in the working directory, where '?' is 1, 2, 3, or 4.
		fastarefprotein10.fasta_part?.fa
		fastarefprotein10.fasta_part?.fa_control.txt
		fastarefprotein10.fasta_part?.fa_pipeline_log.txt
		fastarefprotein10.fasta_part?.fa_pipeline_output.txt
