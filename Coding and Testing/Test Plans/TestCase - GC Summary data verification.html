      Short Title : Summary Table Calculation
      Title : Testing Summary Table Calculations

      VITALS
      Product GeneConnect
      Component Calculation of genomic identifier links
      		 Test Area Calculation of genomic identifier links
      Reference 1 Priority P2

      DESCRIPTION

      		 
      Purpose:
To check whether the Summary tables are correctly populated.

Setup:
1. GeneConnect Server has been deployed. (Please refer GeneConnect Server Readme file for more information).
2. MetaData tables have been populated.
3. All the Base tables have been populated or should be populated as a part of this server run.

Procedure:
1. Configure and Verify the file server.properties in \config folder.
2. Configure and Verify the file CommandFile.xml in \config folder, if any base tables have to be populated as part of this server run.
3. Run the GC Server using "Run" target of Build.xml.
OR 
3. Run "calculateSummary" target of GC server ant build script. (Please refer GeneConnect Server Readme file for more information).


Summary Table Verification Procedure:

1.To verify that the tables GENE, MRNA and PROTEIN are correctly loaded:

(i)Consider the table GENE, this table must have Ensembl Gene ID, Entrez Gene ID and Unigene ID columns. All the ID's in a particular row must belong to a single locus.
Similarly, in the table MRNA (with columns Ensembl Transcript ID, GenBank mRNA ID ,RefSeqmRNA ID) and in table PROTEIN (with columns Ensembl Peptide ID , RefSeq Protien ID, UniprotKB ID and GenBank Protein ID) in a single row must belong to a single locus.

(ii)The correctness of the data loaded can be checked by manually browsing through the Online database (explained in Verification Logic). 

(iii)Each of the tables GENE, MRNA and PROTEIN must have GENE_ID,MRNA_ID and PROTEIN_ID as a unique identifier for a particular row.

2.To verify that the tables GENE_MRNA,MRNA_PROTEIN and PROTEIN_GENE are correctly populated:

(i)In the table GENE_MRNA, the GENE_ID from the GENE table and the MRNA_ID from the MRNA table are related and each such relation/association is shown by a particular row.
Thus if GENE_ID 1 from GENE table is related to MRNA_ID 1 and 2 in MRNA table, then GENE_MRNA table must have the following two rows:
GENE_ID	MRNA_ID
1	1
1	2

(ii)This indicates that all the Genomic Identifiers in the 1st row of GENE table and that of the 1st row of MRNA table belong to the same locus.
Since GENE_ID 1 is also related to MRNA_ID 2, all the Genomic Identifiers in the 1st row of GENE table and that of the 2nd row of MRNA table belong to the same locus.

Similar logic applies to MRNA_PROTEIN and PROTEIN_GENE also.

3.To verify that the table  GENOMIC_IDENTIFIER_SET: is correctly populated:
It has the columns :SET_ID, GENE_ID, MRNA_ID, PROTEIN_ID, SOLUTION_ID
The SET_ID is the unique identifier for all the rows in this table and indicates the association between GENE_ID, MRNA_ID, PROTEIN_ID from the tables GENE, MRNA and PROTEIN respectively.

For e.g. if the table GENOMIC_IDENTIFIER_SET has following data:
SET_ID	GENE_ID	MRNA_ID	PROTEIN_ID  SOLUTION_ID
1	1	1	1	    null
2	1	2	1	    null

For SET_ID 2, 
It shows that GENE_ID 1, MRNA_ID 2 and PROTEIN ID 1 are associated.
This can be verified as follows:
(i)Check in table GENE_MRNA, whether GENE_ID 1 and MRNA_ID 2 are related,i.e. present in the same row.
(ii)Check in table MRNA_PROTEIN, whether MRNA_ID 2 and PROTEIN_ID 1 are related,i.e. present in the same row.
(iii)Check in table PROTEIN_GENE, whether PROTEIN_ID 1 and GENE_ID 1 are related,i.e. present in the same row.

If all the three (GENE_ID,MRNA_ID and PROTEIN_ID) are found to be related then only the data mentioned in the GENOMIC_IDENTIFIER_SET table is correct.

4. To verify that the table SET_ONT is correctly populated:
(i)If a particular SET_ID involves the datasources as follows:
Ensembl Gene ?Entrez Gene-RefSeqmRNA-RefSeqProtein
For this set of DataSources the possible ONT?s would be:
Ensembl Gene ?Entrez Gene-RefSeqmRNA-RefSeqProtein
RefSeqProtein- RefSeqmRNA- Entrez Gene- Ensembl Gene
RefSeqmRNA- Entrez Gene- Ensembl Gene
etc.

(ii)Thus in table SET_ONT, for that SET_ID, all the PATH_ID  (PATH_ID from table ONT ) corresponding to all the paths mentioned above must be present.

Input/Data:
1.All the base tables and MetaData tables must be poplulated.

Expected Output:
All the following tables are populated with correct data:
GENE
MRNA
PROTEIN 
GENE_MRNA 
MRNA_PROTEIN 
PROTEIN_GENE
GENOMIC_IDENTIFIER_SET
SET_ONT

Verification Logic:

1. For a particular SET_ID from GENOMIC_IDENTIFIER_SET , find out the GENE_ID,MRNA_ID and the PROTEIN_ID found to be associated .

2.For the GENE_ID (e.g.: 1) find the actual genomic identifiers in the table GENE corresponding to each of the Data Sources viz., Ensembl Gene ID, Entrez Gene ID and Unigene ID.

3.Go to the page http://www.ensembl.org/index.html. For the Ensembl Gene ID , ( as mentioned in the table GENE) find the links to Entrez Gene and Unigene . Check whether the genomic identifiers mentioned here and those in the GENE table are same for Entrez Gene and Unigene.

Similar logic must be applied for MRNA and PROTEIN also.
This verifies whether the data in the tables GENE,MRNA and PROTEIN are correctly loaded.

4.For the MRNA_ID (e.g.: 2) find the actual genomic identifiers in the table MRNA corresponding to each of the Data Sources viz., Ensembl Transcript, GenBank accession and RefSeqmrna.

5.Go to the page http://www.ensembl.org/index.html. For the Ensembl Gene ID , find the links to Ensembl Transcript, GenBank accession and RefSeqmrna. Check whether the genomic identifiers mentioned here and those in the MRNA table are same for Ensembl Transcript, GenBank accession and RefSeqmrna.

Similar logic must be applied for verification of :
Entrez Gene ID to :Ensembl Transcript, GenBank accession and RefSeqmrna.
Unigene ID to :Ensembl Transcript, GenBank accession and RefSeqmrna.
This verifies that all the genomic identifiers in the GENE table are associated with that of MRNA table for a particular SET_ID.

Thus, this verifies that the table GENE_MRNA is correctly loaded.
The tables MRNA_PROTEIN and PROTEIN_GENE must be verified with a similar logic.

      Short Title : Summary Table Calculation for a newly added Data S
      Title : Testing Summary Table Calculations for a newly added Data Source

      VITALS
      Product GeneConnect
      Component Calculation of genomic identifier links
      		 Test Area Calculation of genomic identifier links
      Reference 1 Priority P2

      DESCRIPTION

      		 
      Purpose:
To test whether the newly added Data Source is correctly represented in the relevant Summary Tables.

Setup:
1. GeneConnect Server has been deployed. (Please refer GeneConnect Server Readme file for more information).
2. MetaData tables have been populated. (This can be done by configuring GCGraph.txt in \cofig folder and running the "calculateMetaData" target of ant script. Refer GC server ReadMe for more details).
3. Base tables for newly added data source have been populated or should be populated as a part of this server run.

Procedure:
1.Configure and Verify  the file server.properties in \config folder.
2. Run the GC Server using Run target of Build.xml.
OR 
3. Run "calculateSummary" target of GC server ant build script. (Please refer GeneConnect Server Readme file for more information).


Input/Data:
1. All the base tables and MetaData tables must be poplulated .

Expected Output:
1.All the following tables are populated with correct data:
GENE
MRNA
PROTEIN 
GENE_MRNA 
MRNA_PROTEIN 
PROTEIN_GENE
GENOMIC_IDENTIFIER_SET
SET_ONT
2. Depending upon the Class of the DataSource , MRNA or GENE or PROTEIN table must have data for the newly added  DataSource. 

Verification Logic:

1. If the DataSource added is a mRNA DataSource (e.g NEW_MRNA), then the table MRNA must have a column for the Data Source NEW_MRNA . The identifier for  NEW_MRNA in a particular row must represent the identifier for the same locus as indicated by the other Genomic Identifiers in that row.

2. For a particular SET_ID from GENOMIC_IDENTIFIER_SET , find out the MRNA_ID/PROTEIN_ID/GENE_ID (Here, MNRA_ID) found to be associated .

3. For the MRNA_ID find the actual genomic identifiers in the table MRNA corresponding to each of the Data Sources viz., Ensembl Transcript, GenBank accession , RefSeqmrna and NEW_MRNA. Check if the Genomic Identifier for NEW_MRNA is present.

4. Verify from the website if all the above four genomic identifiers belong to the same locus. This can be done by browsing the website of one DataSource (e.g. Ensembl) and finding the links to other three DataSources.
