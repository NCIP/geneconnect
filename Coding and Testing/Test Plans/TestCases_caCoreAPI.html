      Test Case ID : 1870
      Short Title : Basic Genomic ID Search
      Title : Basic Genomic ID Search

      VITALS
      Product GeneConnect
      Component caCore API
      		 Test Area Search
      Reference 1 Priority P1

      DESCRIPTION

      		 
      Purpose:

To use caCore Like API query interface to retrive a set of associated genomic identifiers with a given data source as output and genomic identifer as input.

Setup:

1. caCore-Like system has been deployed
2. Application Server (where caCore-Like system is deployed) has been   
   started.
3. For test client refer to:
   http://cabigcvs.nci.nih.gov/viewcvs/viewcvs.cgi/geneconnect/Coding% 
   20and%20Testing/GeneConnectWeb/GeneConnect_caCore_Client/
4. For this test case refer to code 
   GeneConnect_caCore_Client/edu/wustl/geneconnect/testCase  
   /SimpleSearchTest.java

Procedure:
1. To start with create client application.

2. Create a domain object of GenomicIdentifierSet class.

3. Create a domain object of Gene class.

4. Set ensemblGeneId attribute of Gene to 'ENSG00000120738'.

5. Set unigeneAsOutput attribute of Gene to 'true'.

6. Set gene attribute of GenomicIdentifierSet to above creted Gene.

7. Create a domain object of Protein class.

8. Set ensemblPeptideAsOutput attribute of Protein to 'true'.

9. Set protein attribute of GenomicIdentifierSet to above creted Protein.

10. Create a domain object of MessengerRNA class.

10. Set ensemblTranscriptAsOutput attribute of MessengerRNA to 'true'.

11. Set messengerRNA attribute of GenomicIdentifierSet to above creted MessengerRNA.

12. Call a ApplicationService.search(targetClassName,GenomicIdentifierSetObject) of caCore API.

6. Execute the client application.


Input/Data:

The  query object must specify the search value of one or more genomic identifiers and ouyput data sources(attribute of Gene,MessengerRNA and /or Protein class).

Expected Output:

1. A list of objects which satisfy the search condition(s) will be returned. OR An empty list will be returned if no object matches search condition(s).
2. For data list please refer to  caCORE_API_Results.xml file 
   http://cabigcvs.nci.nih.gov/viewcvs/viewcvs.cgi/geneconnect/Coding%
   20and%20Testing/Test%20Activity%20Logs/caCORE_API_Results.xml



Verification Logic:

1. Verify by browsing mannualy through database.  

2. The correctness of the Frequency value for a particular genomic identifier can be verified as follows:
   Frequency is calculated as the ratio of number of occurrences of a ?single genomic identifier? in each combination of genomic identifiers to the total number of traversable paths for the given input.  

3. The correctness of the Confidence Score for a set of genomic identifiers can be verified as follows:
   Confidence score is calculated as the ratio of number of occurrences of a unique ?combination of genomic identifiers (includes only selected input and output data sources)? in a query, to the total number of traversable paths for the given input.
